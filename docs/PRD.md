## 一、原始需求复述

面向个人用户的轻量物联网云端管理系统。终端硬件为低性能红外人体感应设备，仅在本地完成基础人体检测，检测到状态变化后通过 WiFi 上报云端；云端负责数据存储、逻辑运算、告警推送，用户通过 Web 面板管理设备、查看日志、配置通知规则。

**关键约束**：终端为红外感应，无摄像头、无人脸相关功能，无需开发任何图像识别、人脸比对逻辑。

## 二、产品目标

| # | 产品目标 | 衡量标准 |
|---|---------|---------|
| G1 | **可靠的实时告警闭环**：从红外设备检测到人体到用户收到告警通知，端到端延迟 ≤ 5 秒，告警零漏报 | 端到端延迟 P95 ≤ 5s；告警送达率 ≥ 99.5% |
| G2 | **零门槛的设备接入体验**：用户无需网络知识，输入激活码即可完成设备绑定并开始使用 | 新用户从注册到首台设备激活完成 ≤ 3 分钟；激活成功率 ≥ 99% |
| G3 | **清晰的商业化分层**：免费用户使用邮箱告警基础功能，付费用户解锁 QQ 机器人通知等增值能力，形成免费→付费的自然转化路径 | 免费用户→付费转化漏斗可追踪；付费功能开通流程 ≤ 2 步 |

## 三、用户故事

### 普通用户角色

| 编号 | 用户故事 |
|------|---------|
| US-1 | 作为一名**个人用户**，我希望通过邮箱注册账号并通过验证码验证身份，以便安全地拥有独立的设备管理空间。 |
| US-2 | 作为一名**已注册用户**，我希望输入管理员提供的激活码来绑定红外感应设备，以便设备自动关联到我的账号并开始上报数据。 |
| US-3 | 作为一名**设备所有者**，我希望在控制台一目了然地看到所有设备的在线状态和最近告警，以便快速掌握居家/办公区域的安全状况。 |
| US-4 | 作为一名**设备所有者**，我希望为每台设备单独配置防抖间隔和通知渠道，以便区分不同场景（如仓库 vs 办公室）的告警灵敏度需求。 |
| US-5 | 作为一名**付费用户**，我希望开通 QQ 机器人通知后，告警消息能实时推送到我的 QQ，以便在不方便查看邮箱时也能第一时间获知。 |
| US-6 | 作为一名**用户**，我希望在告警历史日志中按时间、设备筛选并查看完整事件记录，以便事后追溯和排查误报。 |

| US-8 | 作为**管理员**，我希望查看用户列表、管理会员权限和订单记录，以便掌握平台运营状况和处理用户问题。 |

## 四、需求池（Requirements Pool）

> 优先级说明：**P0** = Must have（上线必须完成） / **P1** = Should have（重要，上线后尽快迭代） / **P2** = Nice to have（可选增强）

### P0 — 必须完成

| 编号 | 标题 | 描述 | 验收标准 |
|------|------|------|---------|
| REQ-001 | 用户注册（邮箱验证码） | 用户使用邮箱注册账号，系统发送 6 位数字验证码到邮箱，用户输入验证码完成验证后设置密码完成注册 | ① 验证码 5 分钟内有效 ② 同邮箱 60s 内不可重复发送 ③ 注册成功后自动登录并跳转控制台 |
| REQ-002 | 用户登录 | 用户使用邮箱+密码登录，登录页面接入 Cloudflare Turnstile 人机验证 | ① 密码错误连续 5 次锁定账号 15 分钟 ② 登录态有效期 ≥ 7 天 ③ Turnstile 验证通过后才可提交 |
| REQ-003 | 忘记密码（邮箱重置） | 用户通过邮箱验证码重置密码，重置页面接入 Turnstile | ① 验证码与注册验证码复用发送逻辑 ② 重置后旧登录态全部失效 ③ 新密码须满足强度要求（≥8 位，含字母+数字） |
| REQ-004 | 密码安全存储 | 用户密码使用 bcrypt 加盐哈希存储，禁止明文存储 | ① salt rounds ≥ 10 ② 数据库中无明文密码 ③ 接口返回不包含密码字段 |
| REQ-005 | 接口参数校验与安全防护 | 所有 API 接口做入参校验（类型、长度、格式），防止 SQL 注入、XSS 等 | ① 所有入参经校验中间件过滤 ② SQL 使用参数化查询 ③ 敏感接口有速率限制 |
| REQ-006 | 激活码生成与绑定 | 管理员后台生成激活码；用户在前端输入激活码绑定设备，设备上报数据时携带激活码进行身份校验 | ① 激活码为唯一字符串，支持批量生成 ② 一个激活码仅可绑定一个用户一台设备 ③ 已绑定的激活码不可重复使用 |
| REQ-007 | 设备数据上报（状态变化触发） | 设备仅在「无人→有人」状态变化时通过 HTTP 上报云端，携带激活码与状态信息 | ① 仅状态变化触发上报 ② 上报接口校验激活码有效性 ③ 无效激活码返回 401 |
| REQ-008 | 服务端防抖与去重 | 服务端内置防抖、去重、锁定时长机制，防止误报与消息轰炸 | ① 同一设备在锁定时长内（默认 30s）的重复上报被去重 ② 防抖间隔、锁定时长支持用户针对单设备独立配置 ③ 超出防抖窗口的合法告警正常处理 |
| REQ-009 | 事件日志存储与查询 | 完整存储设备上线、离线、人体检测告警等事件，用户可在面板查看历史记录 | ① 事件类型包含：上线、离线、人体检测告警 ② 每条记录含时间戳、设备ID、事件类型、详情 ③ 支持按设备、时间范围筛选 ④ 分页查询，默认每页 20 条 |
| REQ-010 | 邮箱告警通知 | 免费用户默认通知渠道，告警触发后即时推送邮件，复用平台统一 SMTP 配置 | ① 告警触发后 10s 内发出邮件 ② 邮件内容含设备名称、告警时间、告警类型 ③ 发送失败有重试机制（最多 3 次） |
| REQ-011 | 设备级通知配置 | 每个设备支持独立配置通知开关、防抖间隔、通知渠道选择 | ① 通知开关默认开启 ② 防抖间隔默认 30s，可调范围 5s-3600s ③ 免费用户仅可选邮箱渠道 |
| REQ-012 | 管理员激活码管理 | 管理员可生成、查看激活状态、禁用激活码 | ① 支持单个/批量生成 ② 可查看激活码列表及绑定状态 ③ 可禁用未使用的激活码 ④ 支持导出为 CSV |

## 五、UI 设计稿要点

### 整体视觉风格

参考**雨云官网（rainyun.com）**风格，核心设计原则：

- **现代简约**：大量留白，信息密度适中，避免视觉拥挤
- **清爽配色**：主色调采用蓝/青色系（参考雨云主色），背景以浅灰/白色为主，强调色用于关键操作按钮
- **层级清晰**：通过卡片（Card）、阴影、圆角建立视觉层级，主内容区与侧边栏分明
- **交互流畅**：页面切换无刷新（SPA），操作反馈即时（Toast/Snackbar），加载状态有骨架屏
- **圆角风格**：卡片圆角 8-12px，按钮圆角 6-8px，输入框圆角 4-6px
- **字体**：系统默认无衬线字体，正文 14px，标题 18-24px，层级分明

```
┌──────────────────────────────────────────────────────┐
│  [Logo] pir-cloud       [用户头像 ▼]               │  ← 顶部导航栏 (64px高)
├──────────┬───────────────────────────────────────────┤
│          │                                           │
│  控制台   │                                           │
│  设备管理  │            主内容区域                      │
│  告警历史  │            (max-width: 1200px, 居中)       │
│  通知配置  │                                           │
│  个人中心  │                                           │
│          │                                           │
├──────────┴───────────────────────────────────────────┤
│            © 2025 pir-cloud  (底部版权信息)             │
└──────────────────────────────────────────────────────┘
```

- 左侧固定侧边栏（宽 220px），含 Logo + 导航菜单 + 当前会员标识
- 顶部导航栏含面包屑/页面标题 + 用户头像下拉菜单（个人中心、退出登录）
- 主内容区域最大宽度 1200px，水平居中，卡片间距 24px

#### 5.2 登录/注册/找回密码页

- 居中卡片式布局，卡片宽度 400px，白底圆角 12px + 柔和阴影
- 顶部 Logo + 标题，下方表单（邮箱、密码/验证码），底部 Turnstile 验证组件
- 登录/注册/找回密码三个页面通过顶部 Tab 或链接切换
- 输入框使用 MUI TextField，带图标前缀（邮箱图标、锁图标）
- 主按钮全宽，蓝色主色，悬停加深
- 背景使用浅灰渐变或微弱几何图案

#### 5.3 控制台概览页

- 顶部：4 个数据卡片（设备总数、在线设备、离线设备、今日告警数），每个卡片含图标 + 数值 + 趋势标签
- 中部：最近告警列表（表格形式），展示设备名称、告警类型、时间、状态，最多 10 条
- 底部（P2）：告警趋势折线图（近 7 天）
- 空状态：无设备时展示引导卡片（"绑定你的第一台设备" + 激活码输入入口）

#### 5.4 设备管理页

- 顶部操作栏：搜索框 + "绑定设备"按钮（弹出激活码输入对话框）
- 设备列表：表格形式，列含设备名称、激活码、在线状态（绿色/灰色圆点）、最后上报时间、操作（详情/配置/删除）
- 设备名称支持内联编辑（点击编辑图标 → 输入框 → 回车保存）
- 设备配置弹窗/抽屉：通知开关（Switch）、防抖间隔（Slider/Input）、通知渠道（Checkbox 组）

#### 5.5 告警历史日志页

- 顶部筛选栏：设备下拉选择 + 日期范围选择器 + 事件类型筛选 + 查询按钮
- 日志表格：时间、设备名称、事件类型（带颜色标签）、详情（可展开）
- 底部分页器：页码 + 每页条数选择（20/50/100）
- 表格行 hover 高亮，告警类型用不同颜色标签区分（告警-红色，上线-绿色，离线-灰色）

#### 5.6 通知配置页

- 设备列表卡片视图，每张卡片展示设备名称 + 当前通知配置摘要
- 卡片内含：通知开关 Switch、防抖间隔显示、已启用渠道标签
- 点击"编辑"打开配置弹窗，与设备管理页配置弹窗一致

#### 5.7 个人中心页

- 上部：用户信息卡片（头像、昵称、邮箱、注册时间、会员状态）
- 中部：会员信息区（当前等级、到期时间、续费/升级按钮）
- 下部：安全设置区（修改密码表单）
- 表单使用 MUI 组件，修改操作有确认与反馈

### 管理员后台页面

#### 5.8 管理员全局布局

- 与用户端布局结构一致，侧边栏菜单替换为：激活码管理、用户管理、订单管理、系统配置
- 顶部导航栏含"返回用户端"入口 + 管理员标识

#### 5.9 激活码管理页

- 顶部操作栏："生成激活码"按钮（弹窗：输入数量 → 批量生成）+ "导出"按钮 + 状态筛选下拉
- 激活码表格：激活码、状态（未使用/已绑定/已禁用，颜色标签）、绑定用户、生成时间、操作（禁用/查看）
- 批量生成后展示结果弹窗，支持直接复制/导出

#### 5.10 用户管理页

- 搜索栏：按邮箱/昵称搜索
- 用户表格：邮箱、昵称、注册时间、设备数、会员状态、操作（查看详情/管理会员）
- 管理会员弹窗：选择会员等级 + 时长 → 确认开通

#### 5.11 订单管理页

- 筛选栏：时间范围 + 订单状态
- 订单表格：订单号、用户邮箱、套餐、金额、支付时间、状态
- 支持导出 CSV

#### 5.12 系统配置页

- 分 Tab 展示：SMTP 配置 / OneBot 配置
- SMTP Tab：host、port、username、password、发件人地址 + "测试发送"按钮（输入邮箱 → 发送测试邮件）
- OneBot Tab：WebSocket 地址、Token + "测试连接"按钮
- 配置表单底部"保存"按钮，保存后有成功 Toast

## 六、信息架构

### 用户端路由

```
/login                          → 登录页
/register                       → 注册页
/forgot-password                → 找回密码页
/                               → 控制台概览（需登录）
```

## 八、风险提示

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| QQ 机器人通知稳定性 | OneBot 协议依赖第三方客户端，可能不稳定 | 加入重试机制 + 降级到邮箱通知 + 日志记录 |
| 邮件送达率 | 免费用户核心通知渠道，送达率影响产品口碑 | 配置 SPF/DKIM/DMARC；监控发送失败率；备选短信通知（P2） |
| 设备上报安全 | 激活码明文传输存在被抓包风险 | 建议 HTTPS 部署；激活码绑定后生成设备 Token 用于后续上报 |
| 高并发上报 | 大量设备同时上报可能压垮服务端 | 上报接口限流；防抖去重前置；异步处理告警推送 |
